# 开发阶段与路线

项目采用**分阶段开发**策略，逐步验证各层技术，降低调试复杂度：

| 阶段 | 目标 | 状态 |
|------|------|------|
| **Phase 1 — 桌面端** | 完成 Electron 壳 + 前端 UI + 本地 JSON 文件读写的全部功能，独立可运行 | 🚧 进行中 |
| **Phase 2 — 手机端** | 基于同一套前端代码，使用 Capacitor 封装为 Android APK，适配文件权限与路径 | ⏳ 待开始 |
| **Phase 3 — 同步验证** | 两端分别在各自同步文件夹中读写，由 Syncthing 同步，验证冲突处理与数据一致性 | ⏳ 待开始 |

> **当前阶段**：Phase 1。只关注 Electron 桌面端的开发与调试，不涉及 Android 构建。手机端和同步验证将在桌面端功能稳定后依次推进。

## 阶段依赖关系

```
Phase 1 (桌面端)
    │
    └── Phase 2 (手机端) —— 依赖 Phase 1 完成的前端代码
            │
            └── Phase 3 (同步验证) —— 依赖两端均可独立运行
```

Phase 2 与 Phase 3 均依赖前序阶段产出的稳定代码。Phase 1 输出的 `src/` 目录（HTML + CSS + JS）将被 Phase 2 直接复用。
